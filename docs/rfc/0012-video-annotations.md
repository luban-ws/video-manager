# RFC-0012: 视频播放时间戳同步与标注

**状态**: 计划中  
**创建日期**: 2026-03-15  
**依赖**: RFC-0003, RFC-0011  
**优先级**: P1

## 摘要

本 RFC 旨在实现视频播放器与 Markdown 编辑器之间的**时间戳同步**功能。
用户可以在播放视频时快速插入当前时间点，点击 Markdown 中的时间戳链接可自动跳转到视频对应位置，实现"所记即所见"的视频标注体验。

## 目标

1. **快捷插入时间戳**: 在编辑 Markdown 时，通过快捷键或按钮插入 `[01:23]` 格式的时间链接。
2. **点击跳转**: 点击预览中的时间链接，视频播放器自动跳转并播放对应时刻。
3. **视觉反馈**: 在编辑器侧边栏或小部件展示当前播放时间点附近的笔记。

## 技术实现细节

### 1. 时间戳解析与渲染
- 扩展 Markdown 解析器（或通过前端正则表达式拦截）处理 `[mm:ss]` 或 `[hh:mm:ss]` 格式。
- 将这些匹配项渲染为带特定类名（如 `.timestamp-link`）的 `<a>` 标签。

### 2. 双向同步
- **编辑器 -> 播放器**: 监听点击事件。如果点击的是时间戳链接，通过 `ref` 调用 `videojs` 的 `currentTime()` 和 `play()` 方法。
- **播放器 -> 编辑器**:（可选）在编辑区高亮显示与当前时间点最接近的笔记行。

### 3. UI 增强
- 在 `MarkdownEditor` 工具栏增加 "插入当前时间" 按钮。
- 全局快捷键（如 `Cmd+Shift+T`）用于快速插入。

## 执行计划

1. **步骤 1**: 在 `MarkdownEditor.tsx` 中增加插入时间戳的逻辑。
2. **步骤 2**: 实现点击预览区域时间链接跳转视频的功能。
3. **步骤 3**: 优化编辑器样式，支持时间戳高亮。

## 验收标准

- [ ] 点击 Markdown 中的时间戳，视频能够准确跳转。
- [ ] 能够通过按钮一键插入当前视频播放时刻。
- [ ] 时间戳格式支持 `[min:sec]` 并且对用户友好。
